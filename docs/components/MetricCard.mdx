## When To Use

- **Dashboard KPIs:** Display key performance indicators like revenue, user counts, or conversion rates.
- **At-a-glance metrics:** Present important numbers with trend context so users can quickly assess performance.
- **Comparison data:** Show how a metric has changed over time with positive/negative trend indicators.

## Anatomy

- **Icon (Optional):** Displayed on the left in a circular container to visually categorize the metric.
- **Label:** Descriptive name of the metric (e.g., "Total Revenue").
- **Action (Optional):** An interactive element (e.g., menu button) positioned top-right.
- **Value:** The primary metric value, displayed prominently.
- **Trend (Optional):** A percentage change indicator with directional arrow. Green for positive, red for negative.
- **Trend Label (Optional):** Comparison context (e.g., "Since last day").
- **Description (Optional):** Additional context displayed below the value.

## Examples

### Basic Metric Card

Display a metric with a label, value, and trend indicator.

```tsx
import { MetricCard } from 'axiom-ui';

export default function App() {
  return (
    <MetricCard
      label="Revenue"
      value="$1.2M"
      trend={12}
      trendLabel="Since last month"
    />
  );
}
```

### With Icon and Action

Add an icon for visual categorization and an action button for additional options.

```tsx
import { MetricCard } from 'axiom-ui';
import { IconCash, IconDotsVertical } from '@tabler/icons-react';
import { ActionIcon } from '@mantine/core';

export default function App() {
  return (
    <MetricCard
      label="Gross Revenue"
      value="$2.4M"
      icon={<IconCash size={20} />}
      action={<ActionIcon variant="subtle"><IconDotsVertical size={16} /></ActionIcon>}
      trend={8.5}
      trendLabel="vs last quarter"
    />
  );
}
```

### Inline Trend Badge

Display the trend as a compact badge next to the value instead of below it.

```tsx
import { MetricCard } from 'axiom-ui';

export default function App() {
  return (
    <MetricCard
      label="Total Users"
      value="8,421"
      trend={70.5}
      trendInline
      description="Active users across all regions"
    />
  );
}
```

## Guidelines

### Do's
- **Use pre-formatted values:** Pass already-formatted strings (e.g., "$1.2M") as the `value` prop.
- **Provide trend context:** Always include a `trendLabel` when showing trend data to explain the comparison period.
- **Group related metrics:** Display multiple MetricCards in a grid layout for dashboard overviews.

### Don'ts
- **Don't overload with data:** Keep each card focused on a single metric. Use separate cards for different metrics.
- **Don't mix inline and block trends:** Be consistent within a set of metric cards.

## Accessibility

- Uses semantic HTML with proper heading hierarchy for labels and values.
- Trend colors (green/red) are accompanied by directional arrow icons, ensuring the information is not conveyed by color alone.
