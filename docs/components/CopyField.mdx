## When To Use

- **Identifiers:** Let users copy project IDs, resource IDs, or reference codes without selecting text.
- **Secrets:** Present API keys and webhook secrets behind a reveal toggle, with a copy action.
- **Connection details:** Surface endpoints and tokens that users paste elsewhere.

**Don't use for:** Editable input (use a `TextInput`), long-form content (use a `Textarea`), or values users never copy (use `Text` or `KeyValueList`).

## Anatomy

- **Field** — A read-only, monospaced input showing the value.
- **Reveal toggle** — An optional eye button to show or hide a masked value.
- **Copy button** — Copies the value and confirms with a check icon.

## Guidelines

### Do's
- **Mask sensitive values.** Use `masked` for keys and secrets so they stay hidden by default.
- **Use a clear label.** Tell users exactly what they are copying.
- **Confirm the copy.** The built-in check feedback reassures users it worked.

### Don'ts
- **Don't make it editable.** The value is meant to be copied, not changed.
- **Don't reveal secrets by default.** Keep `masked` values hidden until the user opts in.
- **Don't truncate silently.** Let long values scroll within the field rather than cutting them off.

## Accessibility

- The copy and reveal controls are focusable buttons with accessible labels.
- Copy feedback is conveyed via a tooltip and an icon change.
- The field is `readOnly`, so its content cannot be edited by keyboard or pointer.
