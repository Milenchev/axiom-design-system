## When To Use

- **Secondary navigation:** Subsection links within a larger section (settings, admin).
- **Filters panel:** Filtering controls for data views (categories, date ranges, tags).
- **Tool panels:** IDE-style panels with contextual tools or property editors.
- **Collapsible layout:** Giving users control over workspace allocation.

**Don't use for:** Primary app navigation (use NavigationBar), overlay panels (use Drawer), or content that changes with every page (put that in the main area).

## Anatomy

- **Container** — Fixed-width vertical panel, typically on the left.
- **Header** — Optional title or collapse toggle.
- **Content** — Navigation links, filter controls, or tool panels.
- **Footer** — Optional bottom section for secondary actions.
- **Collapse toggle** — Button to minimize the sidebar to icons-only.

## Examples

### Basic sidebar

```tsx
import { Sidebar } from 'axiom-ui';

<Sidebar>
  <Sidebar.Header title="Settings" />
  <Sidebar.Content>
    <Sidebar.Link href="/settings/general" active>General</Sidebar.Link>
    <Sidebar.Link href="/settings/security">Security</Sidebar.Link>
    <Sidebar.Link href="/settings/billing">Billing</Sidebar.Link>
  </Sidebar.Content>
</Sidebar>
```

### Collapsible sidebar

```tsx
import { useState } from 'react';
import { Sidebar } from 'axiom-ui';

function CollapsibleSidebar() {
  const [collapsed, setCollapsed] = useState(false);

  return (
    <Sidebar collapsed={collapsed} onCollapse={setCollapsed}>
      <Sidebar.Content>
        {/* Links show icons only when collapsed */}
      </Sidebar.Content>
    </Sidebar>
  );
}
```

## Guidelines

### Do's
- **Keep width consistent.** 240–280px expanded, 48–64px collapsed.
- **Highlight the active item.** Users need to know where they are.
- **Allow collapsing.** Gives users more horizontal space when needed.
- **Group related links.** Use sections or dividers for logical grouping.

### Don'ts
- **Don't put primary content in the sidebar.** It's for navigation and tools, not the main payload.
- **Don't auto-collapse on desktop.** Let users control their workspace.
- **Don't overflow with too many items.** If you have 20+ links, use collapsible groups.
- **Don't put the sidebar on the right for navigation.** Users expect nav on the left.

## Accessibility

- Uses `<aside>` landmark with `aria-label="Sidebar"`.
- Active link has `aria-current="page"`.
- Collapse toggle communicates state via `aria-expanded`.
- All links are keyboard-navigable in DOM order.
- When collapsed, icon-only links retain their accessible names.
