## When To Use

- **Status indicators:** Show the state of an item (active, pending, error, approved, etc.)
- **Category labels:** Tag content with a category (primary, accent1, dark, etc.)
- **Counts:** Display a number alongside navigation items or lists.
- **Workflow states:** Indicate where an item is in a process (submitted, needreview, escalated).

## Semantic Types

Badge supports semantic `type` names that map to the Axiom categorical color palette. This provides a consistent, meaningful color language across all products.

```tsx
<Badge type="success">Approved</Badge>
<Badge type="warning" emphasis="subtle">Pending Review</Badge>
<Badge type="error" emphasis="minimal">Rejected</Badge>
```

### Available Types

| Type | Use Case |
|------|----------|
| `primary` | Default / main action state |
| `secondary` | Supporting category |
| `tertiary` | Tertiary category |
| `info` | Informational messages |
| `success` | Successful / positive states |
| `warning` | Warning / attention needed |
| `error` | Error / failed states |
| `critical` | Urgent / critical states |
| `dark` | Neutral dark emphasis |
| `pending` | Awaiting action |
| `active` | Currently active |
| `inactive` | Disabled / inactive |
| `accent1` / `accent2` / `accent3` | Additional color categories |

Legacy aliases are also supported: `approved`, `available`, `escalated`, `needreview`, `rejected`, `disabled`, `submitted`.

## Emphasis Levels

Each type can be displayed at three emphasis levels:

```tsx
<Badge type="success" emphasis="bold">Bold</Badge>
<Badge type="success" emphasis="subtle">Subtle</Badge>
<Badge type="success" emphasis="minimal">Minimal</Badge>
```

| Emphasis | Mantine Variant | Description |
|----------|-----------------|-------------|
| `bold` (default) | `filled` | Solid categorical color background with contrast-safe text |
| `subtle` | `light` | 15% tinted background with categorical color text |
| `minimal` | `outline` | Transparent background with categorical color border and text |

## Direct Color Mode

You can bypass the type system and use Mantine's `color` and `variant` props directly:

```tsx
<Badge color="axiom-blue" variant="filled">Custom</Badge>
<Badge color="axiom-red" variant="light">Alert</Badge>
```

## Guidelines

### Do's
- **Use semantic types** for status indicators — they provide consistent color meaning across products.
- **Use `bold` emphasis** for primary status badges that need to stand out.
- **Use `subtle` emphasis** for secondary information that should be visible but not dominate.
- **Use `minimal` emphasis** for badges in dense UIs where space is limited.
- **Keep labels short** — one or two words maximum.

### Don'ts
- **Don't use color alone** to convey meaning — always include a text label.
- **Don't mix emphasis levels** for badges that represent the same data dimension.
- **Don't overuse badges** — too many badges in one view reduce their effectiveness.
- **Don't create custom colors** for status meanings that already have a semantic type.

## Accessibility

- Badge is a visual label, not interactive. It renders as a `<div>` by default.
- Text contrast is automatically handled: bright background colors (yellow, chartreuse) use dark text to meet WCAG AA standards.
- Always pair badge color with meaningful text — never rely on color alone to convey status.
- When used alongside interactive elements, ensure the badge is associated with its parent via proximity or `aria-label`.
