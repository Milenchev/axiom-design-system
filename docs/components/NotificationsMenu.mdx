## When To Use

- **Global alerts:** Place it in the `HeaderBar` so users can review activity from anywhere in the app.
- **Activity feeds:** Surface comments, mentions, system events, and status changes in one place.
- **Unread tracking:** Show an unread count badge and let users clear it with a single action.

**Don't use for:** Transient feedback after an action (use a toast `Notification`), blocking confirmations (use a `Modal` or `Dialog`), or full-page activity history (link to a dedicated page).

## Anatomy

- **Trigger** — A bell button with an unread-count `Indicator`. Pass `children` to customise it.
- **Header** — The `title` plus a "Mark all as read" action shown when there are unread items.
- **List** — A scrollable list of notification items, newest first.
- **Item** — An optional icon tile, title, description, relative time, and an unread dot.
- **Empty state** — A friendly message (`emptyLabel`) shown when the list is empty.

## Examples

### Basic notifications menu

```tsx
import { NotificationsMenu } from 'axiom-ui';
import { IconMessageCircle } from '@tabler/icons-react';

const notifications = [
  {
    id: '1',
    title: 'New comment on your post',
    description: 'Jane Doe replied to your update.',
    time: '2m ago',
    icon: <IconMessageCircle size={18} />,
    color: '#0f6eef',
  },
];

<NotificationsMenu
  notifications={notifications}
  onMarkAllRead={() => markAllRead()}
  onSelect={(id) => openNotification(id)}
/>
```

### Empty state

```tsx
<NotificationsMenu notifications={[]} emptyLabel="No new notifications" />
```

## Guidelines

### Do's
- **Order newest first.** Users expect the most recent activity at the top.
- **Keep titles short.** Lead with the action, put context in the description.
- **Provide a clear empty state.** Reassure users when there's nothing to review.
- **Offer "Mark all as read".** Let users quickly clear the unread count.

### Don'ts
- **Don't use it for ephemeral toasts.** Toasts disappear; this is a persistent inbox.
- **Don't paginate inside the dropdown.** Cap the list and link to a full page for history.
- **Don't rely on colour alone** to convey unread state — the unread dot and background reinforce it.

## Accessibility

- The default trigger exposes the unread count through an `aria-label`.
- Each notification is a focusable button; selecting it marks it read and closes the menu.
- The list is keyboard-scrollable via `ScrollArea`.
- Focus returns to the trigger when the menu closes.
