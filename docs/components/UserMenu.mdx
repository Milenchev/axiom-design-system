## When To Use

- **Account access:** Give the signed-in user a single, predictable place to reach profile, settings, and sign-out.
- **Global header:** Place it at the far end of the `HeaderBar` as the canonical account entry point.
- **Identity confirmation:** Show who is signed in via avatar, name, and email.

**Don't use for:** Primary navigation (use `NavigationBar` or `Sidebar`), product switching (use `ProductMenu`), or notifications (use `NotificationsMenu`).

## Anatomy

- **Trigger** — An avatar with the user's name. Shows initials when no image is provided.
- **Header** — The user's avatar, full name, and email at the top of the dropdown.
- **Items** — Account actions, each with an optional icon. A destructive action (sign-out) can be styled as `danger`.
- **Divider** — Optional separator before an item via `withDividerBefore`.

## Examples

### Basic account menu

```tsx
import { UserMenu } from 'axiom-ui';
import { IconUser, IconSettings, IconLogout } from '@tabler/icons-react';

<UserMenu
  name="Jane Doe"
  email="jane@axiom.dev"
  items={[
    { label: 'Profile', icon: <IconUser size={16} /> },
    { label: 'Settings', icon: <IconSettings size={16} /> },
    { label: 'Sign out', icon: <IconLogout size={16} />, danger: true, withDividerBefore: true },
  ]}
/>
```

### With an avatar image

```tsx
<UserMenu
  name="Jane Doe"
  email="jane@axiom.dev"
  avatar="/avatars/jane.png"
  items={[{ label: 'Sign out', danger: true }]}
/>
```

## Guidelines

### Do's
- **Show identity clearly.** Include name and email so users can confirm the active account.
- **Group account actions.** Keep this menu focused on the user (profile, settings, sign-out).
- **Mark sign-out as destructive.** Use `danger` so it reads as a terminal action.

### Don'ts
- **Don't mix app navigation here.** Use the `Sidebar` or `NavigationBar` for moving around the app.
- **Don't overload the list.** A handful of account actions keeps the menu scannable.
- **Don't hide critical settings exclusively here.** Provide a dedicated settings area too.

## Accessibility

- The trigger is a focusable button that opens the menu on click.
- Menu items are keyboard navigable with arrow keys; `Esc` closes the menu.
- The avatar falls back to the user's initials when no image is available.
- Focus returns to the trigger when the menu closes.
