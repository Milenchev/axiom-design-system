## When To Use

- **Multi-product suites:** Let users move between the apps of a platform (analytics, billing, CRM) without leaving the current context.
- **Global navigation:** Place it in the `HeaderBar` next to the user menu as a persistent app launcher.
- **Workspace switching:** Surface a small, recognisable set of destinations behind a single grid icon.

**Don't use for:** Primary in-page navigation (use `NavigationBar` or `Sidebar`), long flat lists of links (use a `Menu`), or a single destination (use a `Button`).

## Anatomy

- **Trigger** — The element that opens the menu. Defaults to a grid icon button; pass `children` to customise it.
- **Header** — A short label describing the grid (`title`).
- **Grid** — Product tiles laid out in a configurable number of `columns`.
- **Product tile** — An icon tile plus the product name. The active product is highlighted.

## Examples

### Basic product switcher

```tsx
import { ProductMenu } from 'axiom-ui';
import { IconChartBar, IconUsers, IconCreditCard } from '@tabler/icons-react';

const products = [
  { id: 'analytics', name: 'Analytics', icon: <IconChartBar size={20} />, color: '#0f6eef' },
  { id: 'crm', name: 'CRM', icon: <IconUsers size={20} />, color: '#04b000' },
  { id: 'billing', name: 'Billing', icon: <IconCreditCard size={20} />, color: '#735920' },
];

<ProductMenu products={products} activeId="analytics" onSelect={(id) => console.log(id)} />
```

### Navigation with links

```tsx
<ProductMenu
  products={[
    { id: 'docs', name: 'Docs', icon: <IconBook size={20} />, href: '/docs' },
    { id: 'app', name: 'App', icon: <IconApps size={20} />, href: '/app' },
  ]}
/>
```

## Guidelines

### Do's
- **Keep the set small.** A product switcher works best with a handful of recognisable destinations.
- **Use distinct icons and colours.** Visual differentiation makes the grid scannable.
- **Highlight the active product.** Pass `activeId` so users always know where they are.

### Don'ts
- **Don't overload it.** If you need dozens of items, use a dedicated navigation page.
- **Don't mix unrelated actions.** This is for switching products, not for settings or sign-out.
- **Don't hide critical navigation here exclusively.** It complements, not replaces, primary navigation.

## Accessibility

- The default trigger is an `ActionIcon` with an `aria-label` derived from `title`.
- Each product tile is a focusable button (or anchor when `href` is provided).
- The menu opens on click and closes after a selection or on outside click.
- Focus returns to the trigger when the menu closes.
